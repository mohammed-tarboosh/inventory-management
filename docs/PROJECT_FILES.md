# دليل ملفات المشروع

هذا المستند يشرح ملفات المشروع الحالية، ماذا يفعل كل ملف، وكيف يشارك في عمل التطبيق. الهدف منه أن يكون خريطة سريعة لأي شخص يريد فهم البنية بدون فتح الشيفرة كلها.

## كيف يعمل المشروع باختصار

التطبيق مبني على TanStack Start وTanStack Router مع React Query وSupabase. المسار العام هو:

1. الصفحة تدخل عبر `src/routes/*`.
2. `src/routes/__root.tsx` يهيئ الغلاف العام، الترجمة، التنبيهات، ومزامنة الجلسة.
3. الصفحات المؤمنة تمر عبر `src/routes/_authenticated.tsx` الذي يتحقق من تسجيل الدخول.
4. الوصول إلى البيانات يتم عبر عميل Supabase في `src/integrations/supabase/client.ts`.
5. العمليات الإدارية والخاصة بالخادم تمر عبر server functions في `src/lib/api/*` أو عبر الكود الخادم في `src/server.ts` و`src/start.ts`.
6. التنسيق، الترجمة، الصلاحيات، والتصدير إلى PDF/Excel تتم عبر ملفات `src/lib/*` والمكونات المشتركة في `src/components/*`.

Package manager:

- This project now standardizes on **npm**. Run `npm install` to install dependencies and `npm run dev` to start the dev server. Bun lock/config files were removed from the repo.

## ملاحظات سريعة

- ملف `docs/ROUTES.md` الحالي لا يذكر صفحة `audit-logs` رغم وجودها الآن في `src/routes/_authenticated/audit-logs.tsx`.
- `docs/ENVIRONMENT.md` و`src/integrations/supabase/client.ts` يوضحان أن بعض القيم تُقرأ من `VITE_` للواجهة ومن `process.env` للخادم.
- `src/routes/__root.tsx` يحدد `dir="rtl"` افتراضيًا، لذلك الواجهة العربية هي الوضع الأساسي.
- `src/routes/_authenticated.tsx` هو نقطة الحماية الأساسية للصفحات الداخلية.

إذا أردت، أستطيع في الخطوة التالية تحويل هذا المستند إلى نسخة أدقّ أكثر بجدولين إضافيين:

1. جدول يربط كل صفحة داخل `src/routes/_authenticated/` بالمهمة الفعلية التي تنفذها داخل واجهة النظام.
2. جدول يشرح كل ملف من ملفات `src/components/ui/` مع مثال استخدام سريع.
