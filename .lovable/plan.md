# خطة بناء واجهات نظام المخزن

البنية التحتية (قاعدة البيانات، الصلاحيات، i18n، RTL) جاهزة. هذه الخطة تبني كل الواجهات.

## 1. المصادقة والحماية
- `src/routes/login.tsx` — تسجيل دخول باسم المستخدم/كلمة المرور (تحويله داخلياً إلى `username@inv.local`)
- `src/routes/signup.tsx` — إنشاء أول مستخدم (Admin تلقائياً)، ثم يخفى لاحقاً وتُدار الحسابات من صفحة المستخدمين
- `src/routes/_authenticated.tsx` — Layout مع حارس `beforeLoad` يتحقق من `supabase.auth.getUser()` ويوجه إلى `/login`
- `src/lib/usePermissions.ts` — Hook يجلب صلاحيات المستخدم الحالي ويوفر `can(permKey)`
- مكون `<RequirePermission perm="...">` لإخفاء/تعطيل العناصر

## 2. التخطيط الرئيسي والتنقل
- `src/components/AppSidebar.tsx` — قائمة جانبية shadcn مع روابط لكل الأقسام، تظهر فقط ما يملك المستخدم صلاحيته
- Header: زر تبديل اللغة (ع/EN)، اسم المستخدم، تسجيل خروج
- دعم RTL/LTR تلقائي حسب اللغة

## 3. صفحات إدارة البيانات الأساسية
- `/items` — جدول الأصناف مع البحث/الفلترة بالمجموعة، نموذج إضافة/تعديل (اسم ع/EN، كود، مجموعة، وحدة، ملاحظات)، عرض الرصيد الحالي من `item_stock`، استيراد/تصدير Excel، حذف
- `/categories` — شجرة المجموعات مع parent_id، إضافة/تعديل/حذف
- `/units` — وحدات القياس
- `/suppliers` — الموردون
- `/customers` — العملاء
- كل جدول: ترقيم صفحات، بحث، تصدير Excel

## 4. فواتير المشتريات
- `/invoices` — قائمة الفواتير (رقم، تاريخ، مورد، عملة، الإجمالي، الدفع)
- `/invoices/new` و `/invoices/$id` — نموذج فاتورة:
  - رأس: رقم، تاريخ، مورد، عملة، سعر الصرف (يُحمّل من آخر سعر صرف لليوم)، نوع الدفع
  - بنود: إضافة أصناف ديناميكياً (صنف، كمية، سعر أجنبي، يُحسب السعر المحلي = سعر×صرف)
  - حساب الإجماليات تلقائياً
  - الحفظ يُنشئ الفاتورة + البنود → triggers DB تنشئ stock_movements وتحدث آخر سعر شراء
  - زر طباعة الفاتورة (PDF)

## 5. حركة الأصناف والتقارير
- `/movements` — Timeline لحركة الأصناف:
  - فلاتر: صنف معين / مجموعة / فترة (من-إلى) / نوع الحركة
  - عرض: تاريخ، صنف، نوع (شراء/تعديل)، كمية، سعر، مرجع الفاتورة، من أنشأ
  - تصدير Excel/PDF
- `/reports` — لوحة تقارير:
  - تقرير الجرد الحالي (item_stock + قيمة المخزون)
  - تقرير المشتريات حسب الفترة/المورد
  - تقرير حركة صنف
  - تقرير حسب المجموعة
  - كل تقرير قابل للتصدير Excel/PDF

## 6. الديون وكشف الحساب
- `/debts` — قائمة العملاء مع الرصيد الحالي من `customer_balances`
- `/debts/$customerId` — كشف حساب تفصيلي:
  - إضافة عملية: نوع (دين/قبض)، تاريخ، عملة، مبلغ، سعر صرف، رقم فاتورة مرجعية، ملاحظات
  - عرض الحركات مرتبة بالتاريخ مع رصيد متراكم
  - الإجمالي والواصل والمتبقي
  - طباعة/تصدير Excel/PDF

## 7. إدارة المستخدمين والصلاحيات
- `/users` — قائمة المستخدمين (للمدير فقط: `users.manage`)
  - إنشاء مستخدم جديد (server function يستخدم `supabaseAdmin.auth.admin.createUser`)
  - تفعيل/تعطيل، تعديل الاسم
- `/users/$id/permissions` — تخصيص صلاحيات لمستخدم (تفصيلي + مجموعات)
- `/permission-groups` — إدارة مجموعات الصلاحيات (admin/manager/data_entry/viewer جاهزة + قابلة للتعديل)
- Checkbox لكل مفتاح صلاحية مع تسمية عربية/إنجليزية

## 8. الإعدادات
- `/settings` — للمستخدمين بصلاحية `settings.manage`:
  - إدارة العملات (YER, USD, SAR + إضافة)
  - أسعار الصرف اليومية
  - الوحدات

## 9. الاستيراد من Excel
- في صفحة `/items`: زر استيراد يفتح dialog
- قراءة xlsx → تحقق Zod (اسم ع، كود، مجموعة، وحدة) → معاينة الأخطاء → استيراد دفعي
- قالب Excel جاهز للتنزيل

## 10. PWA
- `vite.config.ts`: إضافة `vite-plugin-pwa` مع manifest عربي
- أيقونات التطبيق، خط Cairo مضمن
- إمكانية التثبيت على الجوال والديسكتوب

## التفاصيل التقنية

**خادمية:**
- جميع كتابات الفواتير عبر `createServerFn` + `requireSupabaseAuth` لضمان `created_by`
- إنشاء/حذف المستخدمين عبر `createServerFn` يستخدم `supabaseAdmin`
- استعلامات القراءة عبر supabase client مباشرة من المكونات (RLS تحمي)

**TanStack Query:**
- `queryOptions` لكل جدول، `useSuspenseQuery` في المكونات
- `invalidateQueries` بعد كل mutation

**التصدير:**
- Excel: `xlsx` (موجود)
- PDF: `jspdf` + `jspdf-autotable` (موجود) مع تسجيل خط Cairo لدعم العربية

**RTL:**
- `dir="rtl"` على html حسب اللغة
- جميع جداول shadcn تدعم الاتجاهين
